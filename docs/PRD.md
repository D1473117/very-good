# 隨便吃什麼都好系統 - 產品需求文件 (PRD)

## 1. 專案概述
### 背景與動機
現代人面臨嚴重的「選擇困難症」，每到用餐時間，「要吃什麼？」成為最耗時且令人煩躁的問題。本專案旨在透過開發一個隨機推薦平台，將尋找餐廳與餐點的「決策成本」降至最低，讓選擇變得快速、有趣且無壓力。

### 目標用戶
- **上班族與大學生**：每天需要在有限時間內決定午晚餐，追求效率。
- **群體決策者**：朋友或同事聚餐時，為了避免意見分歧而需要客觀的決策工具。
- **喜愛探索者**：喜歡嘗試新口味、不想侷限於固定選項的美食愛好者。
- **生活極簡主義者**：希望將精力保留給更重要的事情，不想在日常瑣事上花費腦力。

### 核心價值主張
提供極速、直覺且有趣的餐飲隨機推薦服務，終結「隨便」、「都好」、「看你」的無效溝通，拯救每一頓飯的選擇困難。

## 2. 功能需求

### 1. 單人隨機推薦 (一鍵抽籤)
- **使用者故事**：作為一個不知道要吃什麼的上班族，我希望輸入簡單條件（如預算、距離、種類）後能一鍵獲得推薦餐廳，以便我能立刻出發用餐。

### 2. 群組決策模式 (多人轉盤)
- **使用者故事**：作為一個群體決策者，我希望可以建立一個群組房間讓朋友加入，大家一起轉盤或隨機挑選，以便公平且快速地決定聚餐地點，避免沒人做決定的尷尬。

### 3. 自訂口袋名單 (專屬題庫)
- **使用者故事**：作為一個生活極簡主義者，我希望可以建立並管理我自己的「口袋名單」，並且只從這個名單中隨機抽取，以便確保我吃到的都是我信任且有興趣的餐廳。

### 4. 飲食避雷針 (黑名單與條件排除)
- **使用者故事**：作為一個有特定飲食偏好或限制的人，我希望可以設定「今天不想吃」或「過敏/忌口」的選項（如：不吃辣、不吃麵、避開素食），以便系統推薦時能自動過濾這些地雷。

### 5. 探險歷史紀錄與評價
- **使用者故事**：作為一個喜愛探索者，我希望系統能記錄我過去隨機抽中的餐廳與我的簡單評分，以便我回顧我的美食探險軌跡，並作為未來調整口袋名單的參考。

## 3. 非功能需求
### 技術限制
- 後端框架：使用 Flask 進行輕量級開發。
- 模板引擎：使用 Jinja2 渲染前端頁面。
- 資料庫：使用 SQLite 儲存用戶資料、餐廳清單與歷史紀錄。

### 效能與安全考量
- **效能**：隨機抽取的演算法必須在 1 秒內回傳結果，以符合「降至最低決策成本」的目標。
- **安全**：若包含使用者註冊與登入功能，需確保密碼加密儲存，並防範基本的 SQL Injection 攻擊。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必要功能)
- 單人隨機推薦 (一鍵抽籤，內建預設餐廳庫)。
- 基礎的條件篩選（如：飯/麵、中式/西式）。
- 簡單響應式網頁介面 (手機/電腦皆可操作)。

### Should Have (重要功能)
- 自訂口袋名單（使用者可以新增自己的餐廳）。
- 飲食避雷針 (設定不想吃的種類)。

### Nice to Have (加分功能)
- 群組決策模式 (分享連結一起抽)。
- 探險歷史紀錄與評價。
- 與 Google Maps API 串接，直接顯示店家資訊與導航。
